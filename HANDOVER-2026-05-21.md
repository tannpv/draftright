## iOS keyboard 401 — FIXED this session (context if it recurs)
Tone rewrite from the keyboard 401'd while the in-app Playground worked.
Cause: the `com.draftright.v2/app_group` MethodChannel was registered in
`AppDelegate.didInitializeImplicitFlutterEngine`, which **never fires** on
this app's UIScene lifecycle, so the keyboard's App Group never got the token
or backend URL. **Fixed** by registering the channel in
`SceneDelegate.scene(_:willConnectTo:)` (commit `7f9a7693`).
Gotcha: `simctl spawn … defaults read group.…` shows a stale cfprefsd cache —
read the App Group plist file directly to verify.
